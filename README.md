# Review SaaS — AI 리뷰 관리 솔루션

> **한 줄 요약**: 월 100만원짜리 플레이스 운영대행을 월 5만원 SaaS로 대체한다.

---

## 🎯 사업 컨셉

### 적용 공식
1. 매달 큰 돈(50~200만원) 쓰는 시장을 찾음 → **플레이스 운영대행 / 리뷰 관리 대행**
2. 동일 가치를 1/20 가격(월 4.9~9.9만원)에 제공하는 SaaS를 만듦
3. 직접 방문 데모 → 한 매장씩 전환
4. 같은 사장님의 다른 매장 → 같은 상권 사장님 소개로 확장

### 포지셔닝
> **"플레이스 운영대행을 월 5만원으로"**
> 사장님이 직접 운영하되, AI가 운영대행사 수준의 인사이트를 매주 제공한다.

---

## 📊 시장 조사 핵심 발견

### 발견 1: 단순 답글 자동화는 레드오션
- 경쟁사: 딜리봇, 리본오토, 댓글몽, 브이리뷰, 크리마
- "AI 답글 자동 생성"만으로는 차별화 불가능

### 발견 2: 네이버 2026년 정책 변화 (★중요)
- 비슷한 문장 구조의 리뷰/답글은 **마이너스 요인**
- 잘 만들어진 플레이스 < **계속 관리되는 플레이스** 우선 노출
- → 단순 자동 답글은 오히려 노출 하락 위험
- → 기존 자동화 도구가 위험해진 시장 = **우리에게 기회**

### 발견 3: 핵심 페인은 "답글"이 아님
- 노출 핵심: 사진(내부/외부/메뉴/가격), 체류시간, 전화/길찾기/예약 클릭률
- 사장님이 진짜 돈 쓰는 건 "플레이스 마케팅 대행" 패키지 (월 50~200만원)
- 답글은 그 패키지의 일부일 뿐

---

## 🏗️ 제품 구조 — 4개 모듈

### 모듈 1: 리뷰 통합 인박스 (기본기)
- 5개 플랫폼 통합: 네이버 / 배민 / 쿠팡이츠 / 카카오맵 / 구글맵
- **AI 초안 + 사장님 1초 컨펌** 방식 (자동발행 X — 네이버 정책 대응)
- 사장님 톤 학습 (과거 답글 5개 입력으로 시작)

### 모듈 2: 플레이스 헬스 스코어 ★ 차별화 핵심
- 매주 월요일 자동 발송
- 노출수, 클릭수, 전화/길찾기 클릭률
- 경쟁점포 대비 사진 업데이트 빈도
- "2주째 신규 사진 없음 → 노출 하락 위험" 알림
- 부족한 메뉴/가격 정보 자동 감지

### 모듈 3: 위기 대응 (Crisis Alert)
- ⭐1~2점 리뷰 즉시 카톡 알림
- AI가 상황 분류: 음식문제 / 배달문제 / 직원응대 / 명백한 블랙
- 상황별 답글 템플릿 + 삭제 요청 가능 여부 자동 판단

### 모듈 4: 월간 인사이트 리포트
- 리뷰 키워드 트렌드, 메뉴별 만족도
- 메뉴 개선 제안 ("김치찌개 짜다는 리뷰 4건 → 간 조정 검토")
- Excel 자동 생성 후 사장님께 발송

## 🎯 타겟

### 1차 타겟: 매장 2~5개 운영 외식 프랜차이즈 점주
- 1인 운영자: 결제력 약함
- 본사형 프랜차이즈: 영업 사이클 너무 김
- **2~5개 점주: 돈 있음 + 본인 결정 + 페인 명확 + 매장당 결제(×3)**

### 2차 타겟
- 동물병원, 미용실, 학원 (리뷰 민감 + 고객 LTV 큼)

## 💰 가격

| 플랜 | 가격 (매장당/월) | 타겟 |
|---|---|---|
| Lite | 4.9만원 | 1매장 사장님 |
| Pro | 9.9만원 | 다매장 / 위기 대응 필요 |
| Agency | 3.9만원 | 프랜차이즈 본사 (10개+) |

> 대행사 월 100만원 vs 우리 월 5만원 = **20배 저렴, 안 쓸 이유 없음**

---

## 🚀 영업 전략

1. **발견**: 네이버 지도에서 평점 4.0 이하 + 리뷰 50개+ 매장 = 즉시 리드
2. **만듦**: 위 4개 모듈로 월 5만원 솔루션
3. **데모**: 사장님 매장 리뷰 분석 리포트 들고 직접 방문 (이미 만들어진 인사이트로 후킹)
4. **확장**: 1매장 → 같은 사장님 다른 매장 → 같은 상권 소개

**핵심 멘트**:
> "지금 마케팅 대행에 월 100만원 쓰시죠? 이거 5만원이면 됩니다."

## 🛠️ 진행 순서

### ✅ Phase A — 아키텍처 (1주차 전반)
**agent / skill 파일 먼저 작성**
agent와 skill의 책임 경계를 명확히 잡고 시작.

작업 순서:
1. 5개 agent 정의 (organizer / collector / drafter / crisis / reporter)
2. 11개 skill 정의 (플랫폼별 스크래퍼 + 핵심 로직 + 통합)
3. agent ↔ skill 의존 관계 다이어그램 확정
4. 데이터 모델(Review, Store, Reply, HealthScore) 스키마 확정

### ✅ Phase B — MVP 구현 (1주차 후반 ~ 4주차)
**아키텍처 확정 후 바이브코딩 착수**

| 주차 | 산출물 | 검증 기준 |
|---|---|---|
| 1주 | 네이버 + 배민 크롤러 + DB | 본인 단골가게 1곳 데이터 수집 성공 |
| 2주 | AI 답글 초안 + 톤 학습 + 대시보드 | 사장님이 "내가 쓴 거 같다" 반응 |
| 3주 | 헬스 스코어 + 카톡 알림 | 첫 데모 매장 확보 (지인) |
| 4주 | Excel 리포트 + 결제 (토스페이먼츠) | 무료 1개월 → 유료 1건 전환 |
